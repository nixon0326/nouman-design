# NouMan Creative DNA

## ブランドアイデンティティ（最重要）

NouMan は **「未来を予測するAI」ではありません。**

NouMan は **「観察を支えるAI」** です。

この違いがすべての広告の土台になる。

| 言ってよいこと | 言ってはいけないこと |
|-------------|-----------------|
| 「観察を、未来の豊かさに」 | 「AIが農業を変える」 |
| 「現場の違和感を受け止める」 | 「データで収益を最大化」 |
| 「話すだけで記録完了」 | 「高精度AIが判断」 |
| 「あなたの記憶が、畑を守る」 | 「スマート農業の新時代」 |
| 「農家と一緒に考える」 | 「農業DX」「IoT農業」 |

## レイアウト DNA

### 基本原則

- **コピーを最優先** — レイアウトはコピーに従う
- **左揃え** — 日本語は左からの視線フローに合わせる
- **余白を大きく** — 詰め込まない。余白は「呼吸」
- **情報量は最小** — 伝えることは1つ。多くて2つ
- **背景写真は世界観の根拠** — 農場・空・自然光・リアル
- **スマホは証拠** — コピーで言ったことをUIが証明する

### グリッド設計（App Store 縦型）

```
┌─────────────────────────────┐
│  [余白]                      │  ← 上部余白 10〜15%
│                              │
│  ヒーローコピー               │  ← 上部 20〜35%
│  （大・太・左揃え）            │
│                              │
│  サブコピー                  │  ← コピー直下
│  （中・通常・左揃え）          │
│                              │
│  ┌──────────────────┐       │
│  │   スマホ画面       │       │  ← 中下部
│  │  （実際のUI）      │       │
│  └──────────────────┘       │
│                              │
│  [CTA or ブランドメッセージ]  │  ← 下部
└─────────────────────────────┘
```

## 視線誘導 DNA

視線は「重力」に従う。上から下へ。左から右へ。

```
ロゴ（小・コーナー）
        ↓
コピー（大・インパクト）
        ↓
サブコピー（補足）
        ↓
CTA（行動喚起）
        ↓
スマホ（証拠）
        ↓
ブランドメッセージ（余韻）
```

### 視線が止まるポイント

1. **コントラスト** — 白背景 × 黒テキスト OR 暗背景 × 白テキスト
2. **サイズの差** — ヒーローコピーとサブコピーのサイズ比は 3:1 以上
3. **余白** — テキストの周囲に十分な余白があると視線が集中する

## コピー DNA

### 原則

- **短く** — 1行 10〜16文字以内が理想
- **強く** — 動詞で終わる or 体言止め
- **一瞬で理解できる** — 農家が読まなくてもわかる

### 良いコピーの構造

```
[課題の共感] + [解決の予感]
または
[現状の否定] + [新しい未来]
または
[驚き] + [証拠]
```

### NouMan コピー例

| パターン | コピー例 |
|---------|---------|
| 課題共感 | 「なんか葉っぱがおかしい」── その違和感が、答えだった。 |
| 現状否定 | もう、泥だらけの手で入力するのは終わりにしよう。 |
| 未来提示 | 観察を、未来の豊かさに。 |
| 驚き | ただのつぶやきが、自動で記録になる。 |
| 先回り | 天候と記憶から、次の「想定外」を先回り。 |

## 背景写真 DNA

### 使ってよいもの

| カテゴリ | 具体例 |
|---------|-------|
| 農地 | 整然とした畑の畝・マルチシート・ハウス |
| 空 | 青空・雲・夕暮れ・夜明け |
| 自然光 | 朝の光が差し込む農場・逆光の葉 |
| 農家 | 作業中の手元・後ろ姿・農機具（人の顔は避ける） |
| 作物 | 新鮮な野菜・果実・生育中の苗 |

### 使ってはいけないもの

| 禁止 | 理由 |
|------|------|
| CG・3D レンダリング | リアリティがなく信頼感が損なわれる |
| ストックフォトのステレオタイプ農家 | 白衣・満面の笑顔・作り物感 |
| 都会的な風景 | NouMan の世界観と合わない |
| 暗すぎる・汚い農場 | ブランドイメージが損なわれる |
| テキストが多く入った写真 | コピーと競合する |

## ブランドカラーの使い方 DNA

| 色 | Hex | 広告での役割 |
|----|-----|------------|
| Brand Green | `#4C8C5A` | CTA・強調テキスト・アクセントのみ（5〜10%） |
| White | `#FFFFFF` | コピーテキスト・背景（暗背景の場合） |
| Black / Deep | `#000000` / `#1E3A2F` | コピーテキスト（明背景の場合） |
| Beige | `#EAD9B5` | 温かみが必要な背景・セカンダリ |

**Brand Green を背景として大きく使わない。**
緑は農業の色でもあるが、使いすぎると「ただの農業アプリ」になる。

## 禁止パターン DNA

以下は「悪いクリエイティブ」の典型。毎回確認すること。

| 禁止 | 理由 |
|------|------|
| 機能の羅列 | 読まれない・ダウンロードされない |
| 情報量が多い | 視線が迷う・何も伝わらない |
| 複数の CTA | 行動できない |
| ロゴを大きくセンターに | 広告ではなく名刺になる |
| グラデーション | トレンドに見え、時代遅れになる |
| テキストに色を多用 | ブランドが安定しない |
| スマホを中心に大きく | UI のデモ動画になる。広告ではない |
| 英語の内部概念名 | 農家に伝わらない |

### 正解 #06 の解剖

#### ① ヒーローコピー構造

```
[動詞] + だけで、[価値] が [完成/変わる]。
↓
「話すだけで、記録が完成する。」
```

**なぜ強いか:**
- 「だけで」= ハードルが低い（簡単さの保証）
- 「〜が完成する」= 結果が具体的でイメージできる
- 動詞起点 = 体験から入るので農家が「自分ごと」化する

#### ② サブコピー構造

```
[否定による共感] + [現場感の証明]
↓
「入力不要。現場でそのまま。」
```

**なぜ強いか:**
- 「入力不要」= 農家最大の痛点（手が汚れる・時間がない）を真っ先に否定
- 「現場でそのまま」= 農場に立ちながら使えるイメージ
- 体言止め×2 = リズムがよく記憶に残る

#### ③ 機能バッジの設計ルール

白い横長バッジ（pill shape）に：
- Brand Green のアイコン
- 太いメインラベル
- 薄いサブ説明

```
[ 🎤  音声で簡単記録       ]
[    作業も気づきも話すだけ ]
```

**5つの機能の順番:**
```
1. 音声で簡単記録（入口 — 使い方）
↓
2. AI が自動で整理（魔法 — 価値）
↓
3. 天気・病害虫を予測（先読み — 安心）
↓
4. 圃場をまとめて管理（統合 — 便利）
↓
5. 未来の収穫を見える化（成果 — ゴール）
```

→ 「使い方」→「魔法」→「安心」→「便利」→「成果」の順で感情を積み上げる

#### ④ CTA 設計

```
[ 無料ではじめる  › ]  ← Brand Green (#4C8C5A)、横幅広め、左詰め
```

- 「無料」を先に置く（DL への心理的ハードルを最初に消す）
- 「ではじめる」= 軽い行動（始めるだけ）
- 「›」= 先に進む方向性の暗示

#### ⑤ ブランドメッセージ（最下部）

```
農（Nou）に関わる全ての人々（Human）に幸あれ。
```

- 機能説明の後に哲学を置くことで「この会社は信頼できる」という余韻を作る
- 小さく・薄く・主張しない = 「存在に気づかせる」レベル

#### ⑥ スマホの斜め配置

右側に斜め（5〜10度傾き）で配置。

**効果:**
- 平置き = 静的・図鑑的
- 斜め = 動的・手に持っている感・リアリティ
- 奥行きが生まれ、背景の農場と自然に馴染む

### 正解 #06 から導いた新ルール

**ルール CD-01: 「〜だけで、〜が完成する」コピーパターン**
- 内容: 「[最小動作]+だけで、[価値結果]が[完成/変わる/始まる]。」
- 理由: 簡単さ(だけで)と結果(完成する)を同時に証明できる最強構文

**ルール CD-02: サブコピーは否定から入る**
- 内容: 「[痛点の否定]。[現場感の証明]。」
- 理由: 農家は「これは自分の問題を解決するか？」を0.5秒で判断する。最初に否定すると「自分ごと」化が早い。

**ルール CD-03: 機能は5つ以内・感情の積み上げ順に並べる**
- 内容: 「使い方 → 魔法 → 安心 → 便利 → 成果」の順で機能を列挙する
- 理由: 感情の山を作ることで「これだけ価値がある」という確信を積み上げられる

**ルール CD-04: 機能バッジは pill shape（白・横長）**
- 内容: 白背景の横長バッジ + Brand Green アイコン + 太いラベル + 薄いサブ説明
- 理由: 視認性が高く、ブランドカラーを適切な割合で使える

**ルール CD-05: CTA は「無料」を先置き**
- 内容: 「無料で〜 ›」の形式。矢印必須。Brand Green の横幅広めボタン
- 理由: ダウンロードの最大の壁は「お金がかかる？」という不安。先に消す。

**ルール CD-06: スマホは斜め（5〜10度）・右配置**
- 内容: テキスト左・スマホ右斜め。平置き禁止
- 理由: 動きと奥行きが生まれ、農場背景と馴染む。「手に持っている」現場感

**ルール CD-07: ブランドメッセージを最下部に置く**
- 内容: 小さく・薄く・1行の哲学を最下部に配置
- 理由: 機能説明の後に信念を置くと「信頼できる会社」の余韻が残る
