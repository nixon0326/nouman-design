## App Store スクリーンショットの基本仕様

| 項目 | iPhone（推奨） | iPad |
|------|-------------|------|
| 解像度 | 1290×2796px（iPhone 16 Pro Max） | 2048×2732px |
| 形式 | PNG / JPEG |
| 最大枚数 | 10枚（最初の3枚が最重要） |
| テキストオーバーレイ | 可能（Apple 審査あり） |

**最初の3枚で勝負が決まる。** ユーザーは4枚目以降を見ない。

## 世界標準の構成パターン

### パターン A：コピー主体型（推奨）

```
[農場背景写真]
　大きなヒーローコピー
　サブコピー
　　　　　　　　　スマホ（証拠）
```

使用例: Nike Run Club、Duolingo、Airbnb  
効果: 感情移入・ブランド価値の伝達

### パターン B：BEFORE / AFTER 型

```
[BEFORE]              [AFTER]
複雑な入力フォーム  →  話すだけで完了
```

使用例: 多くの生産性アプリ  
効果: 課題解決の明確な訴求  
注意: 「比較」が明確でないと効果半減

### パターン C：機能紹介型（非推奨）

```
[スマホ UI 中心]
機能名
機能説明
```

効果が低い。NouMan には使わない。  
理由: 機能を並べても感情が動かない。

## 世界トップアプリのクリエイティブ法則

### コピーの法則

**法則 1: タイトルは問いかけか体言止め**
- Good: 「観察を、未来の豊かさに。」
- Good: 「もう、面倒な入力は終わりにしよう。」
- Bad: 「NouMan Worker の機能をご紹介します」

**法則 2: コピーは農家の言葉で書く**
- Good: 「なんか葉っぱがおかしい」
- Bad: 「病害虫リスク検知AIシステム」

**法則 3: ベネフィットを先に、機能は後で**
- Good: 「話すだけで記録完了」→（スマホ画面で証明）
- Bad: 「音声入力機能搭載」→「だから何？」

### レイアウトの法則

**法則 4: スマホは右下または中央下**
農作業のイメージを邪魔しない。証拠として添える。

**法則 5: コピーは左上または左中**
日本語は左から右、上から下の視線フロー。

**法則 6: 背景写真は上2/3、スマホは下1/3**
上でエモーション、下でロジック（証拠）。

### 心理の法則

**法則 7: 農家は「自分の話」にしか反応しない**
「農業AI」より「あなたの畑の話」。
「高精度システム」より「疲れた日の記録が簡単」。

**法則 8: 信頼は積み重ねで生まれる**
1枚目で止まらせる → 2枚目で共感 → 3枚目で信頼 → DL。
各枚は独立ではなく「ストーリー」。

**法則 9: ダウンロードの壁は「面倒そう」**
「かんたん・すぐ使える・無料」を何らかの形で伝える。

## Google Play ストア固有のルール

| 項目 | 詳細 |
|------|------|
| フィーチャーグラフィック | 1024×500px（必須） |
| スクリーンショット | 最小2枚、推奨8枚 |
| 縦横 | 縦型・横型どちらも可 |
| 動画 | YouTube URL で設定可 |

Google Play では **フィーチャーグラフィック** が最初の印象を決める。  
これは App Store のスクリーンショット 1枚目と同等の重要性を持つ。

---

## App Store 審査における注意事項

| 禁止 | 代替案 |
|------|-------|
| 実在しないデバイス・OS の表示 | 実際のスクリーンショットを使用 |
| 誤解を招く機能の表現 | 実際のアプリ機能のみ表示 |
| 他社サービスとの比較 | 自社の価値のみ訴求 |
| 過大な効果の主張 | 実績・具体的なデータがあれば使用可 |

## チェックリスト（制作前）

App Store クリエイティブを制作する前に確認すること：

- [ ] 最初の3枚がストーリーになっているか
- [ ] 1枚目で「3秒で止まる」か
- [ ] 農家の言葉でコピーが書けているか
- [ ] スマホ画面は実際のアプリ画面か
- [ ] CTA は1つか
- [ ] 解像度は 1290×2796px か
